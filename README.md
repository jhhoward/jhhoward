Hi, I'm James Howard, a systems & video game developer specialising in low-level optimisation, real time graphics, and networking. I have a passion for pushing the limits of hardware ranging from 8-bit / 16-bit vintage computers and microcontrollers up to modern high end platforms. Have a look at some of my personal projects :)

## Featured projects
* [MicroWeb](https://github.com/jhhoward/MicroWeb) (**C++/ASM**) is a highly memory-efficient 16-bit web browser engineered specifically for legacy x86 DOS machines that can run on an IBM XT with 512K of RAM.
* [Faux86](https://github.com/jhhoward/Faux86) (**C++**) is an IBM PC / x86 emulator that runs bare metal on Raspberry Pi boards, no operating system required!
* [Wolfenstein CGA](https://github.com/jhhoward/WolfensteinCGA) (**C/ASM**) is a fork of the 1992 *Wolfenstein 3D* engine, featuring a completely rewritten software renderer targeted at the CGA video card architecture.

## Microcontroller projects [(Arduboy)](https://www.arduboy.com)
* [Catacombs of the Damned!](https://github.com/jhhoward/Arduboy3D) is a 3D first person shooter / dungeon crawler with procedurally generated levels running on an 8-bit microcontroller.
* [MicroCity](https://github.com/jhhoward/MicroCity) is a city builder inspired by SimCity, optimised to run within the Arduboy's 2.5K memory constraint.
* [Wolfenduino FX](https://github.com/jhhoward/WolfenduinoFX) is a demake of Wolfenstein 3D, squeezing the entire shareware episode into the Arduboy FX platform.
* [Ardutosh](https://github.com/jhhoward/Ardutosh) is a custom windowed GUI desktop environment / OS, inspired by early Macintosh systems.
* [ArduSoccer](https://github.com/jhhoward/ArduSoccer) is a 5-a-side football game with networked multiplayer support.
* [Bone Shakers](https://github.com/jhhoward/BoneShakers) is a 'mode 7' style pseudo 3D racing game, inspired by Mario Kart.

## Other fun stuff
* [HTSGET](https://github.com/jhhoward/HTSGET) (**C++**) is a fork of mTCP's HTGET, which adds support for modern HTTPS / TLS transfers using a modified version of wolfSSL, running inside a 16-bit DOS environment.
* [Vidsync](https://github.com/jhhoward/Vidsync) (**x86 ASM**) is a TSR DOS application to fix screen calibration issues with the Book8088 laptop.
* New video drivers for Sierra's SCI adventure game engine: [Written from scratch in x86 ASM](https://github.com/jhhoward/SCIVideoDrivers) and [reverse engineered and binary modified versions of the original drivers](https://github.com/jhhoward/SCIVideoDriverMods).
* [Electron Adventure](https://github.com/jhhoward/ElectronAdventure) (**6502 ASM**) is a graphical adventure game engine with a custom scripting language / VM and text parser, inspired by Space Quest for the Acorn Electron / BBC Micro and written in pure 6502 assembly.
